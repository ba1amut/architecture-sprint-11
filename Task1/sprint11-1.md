**Проблемы**

## 1. Проблемные места

| Проблема | Описание | Влияние |
|----------|---------|--------|
| 1. Устаревшие системы | SQL Server 2008 и Power Builder замедляют работу и мешают масштабированию. | Высокое |
| 2. Медленная отчетность | Текущие BI-инструменты перегружены трансформациями, отчеты строятся часами. | Высокое |
| 3. Сложность интеграции | Шина данных Apache Camel устарела, добавление новых сервисов требует больших затрат. | Высокое |
| 4. Управление доступами | Разграничение прав на чувствительные данные (финансы, медицина) недостаточно гибкое. | Среднее |
| 5. Дублирование данных | Несколько копий одних и тех же данных в разных системах ведут к несогласованности. | Среднее |
| 6. Высокая стоимость поддержки | Легаси-системы требуют значительных затрат на поддержку и модернизацию. | Низкое |

## 2. Приоритизация (метод MoSCoW)

- **Must-have:**
    - Миграция с SQL Server 2008 на современное DWH.
    - Оптимизация BI-отчетности через новую "витрину данных".
    - Переход с Apache Camel на Kafka.

- **Should-have:**
    - Внедрение ролевой модели доступа для чувствительных данных.
    - Оптимизация хранилища для устранения дублирования данных.

- **Could-have:**
    - Снижение затрат на поддержку легаси-систем путем их постепенного вывода из эксплуатации.

- **Won't-have (в ближайший год):**
    - Полный отказ от устаревших BI-инструментов (будет постепенный переход).
    - Полное удаление легаси-систем (они останутся в отдельных доменах временно).
