#### **1. Медицинский домен**

- **Что включает**: EHR-систему, данные медоборудования, кэш для быстрого доступа.
    
- **Изменения**:
    
    - Отказ от прямой записи в DWH → данные сначала в кэш (Redis), затем в Kafka.
        
    - Мед. карты исключены из аналитики (остаются только в EHR).
        
- **Преимущества**:
    
    - Клиники получают мгновенный доступ к актуальным данным без нагрузки на DWH.
        
    - Снижение рисков нарушения конфиденциальности медданных.
        

#### **2. Финтех-домен**

- **Что включает**: Банковские операции, платежи, кредитный скоринг.
    
- **Изменения**:
    
    - Прямая интеграция с Data Lake через CDC (Change Data Capture).
        
    - Отдельный API для ИИ-скоринга.
        
- **Преимущества**:
    
    - Соответствие PCI DSS за счёт изоляции финансовых данных.
        
    - Возможность обновлять банковские сервисы без изменений в других доменах.
        

#### **3. ИИ-домен**

- **Что включает**: ML-модели, сервис прогнозов, API для интеграции.
    
- **Изменения**:
    
    - Подписка на события из Kafka (медицина + финансы).
        
    - Публикация прогнозов через API Gateway.
        
- **Преимущества**:
    
    - Модели обучаются на актуальных данных из стримов.
        
    - Возможность A/B-тестирования новых алгоритмов без downtime.
        

#### **4. Аналитический домен**

- **Что включает**: Data Lake (Raw/Cleaned), витрины, BI-инструменты.
    
- **Изменения**:
    
    - Исторические данные из DWH импортируются однократно.
        
    - Все новые данные поступают через Kafka.
        
- **Преимущества**:
    
    - Аналитики работают с согласованными данными из всех доменов.
        
    - Возможность строить кросс-доменные отчеты (например, "Влияние лечения на платежеспособность").
        

#### **5. Легаси DWH**

- **Роль**: Только для исторических данных и legacy-отчетов.
    
- **Изменения**:
    
    - Постепенное отключение от бизнес-процессов.
        
- **Преимущества**:
    
    - Снижение затрат на поддержку устаревшей системы.
        

---

### **Ключевые преимущества новой архитектуры**

1. **Независимость доменов**
    
    - Каждый домен развивает свою логику без согласований с другими.  
        _Пример_: Финтех может обновить платежный шлюз, не затрагивая медицинские сервисы.
        
2. **Гибкость интеграции**
    
    - Новые бизнесы (фармацевтика, электроника) подключаются через Event Bus без переделки DWH.
        
3. **Скорость аналитики**
    
    - Data Lake с витринами дает ответы за секунды вместо часов ожидания отчетов из DWH.
        
4. **Снижение рисков**
    
    - Изоляция персональных (EHR) и финансовых данных 
        
5. **Масштабируемость**
    
    - Kafka и облачный Data Lake выдерживают рост нагрузки в 10-100x.
